# Database & Prisma Context
- **Stack**: PostgreSQL, Prisma ORM.
- **Location**: `packages/db/prisma/schema.prisma`.
- **Tenancy Rules**: 
  - Almost all models have a `tenantId`.
  - Queries MUST scope by `tenantId`: `prisma.message.findMany({ where: { tenantId, ... } })`.
- **Migrations**: 
  - Never modify the database directly. Use `npx prisma migrate dev`.
  - Avoid destructive migrations (dropping columns) without a multi-phase rollout.
- **Performance**:
  - Always `.select()` only necessary fields.
  - Use `@@index` on frequently queried columns (`tenantId`, `createdAt`).
